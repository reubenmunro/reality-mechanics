# Method Reconciliation — Mechanic -> Places

This is not a new methodological layer.

It is a reconciliation between the working recovery language and the Atlas's existing note grammar.

## Background

The recovery method temporarily used the term Mechanic to describe the point at which a dependency structure became independently describable.

Repeated testing across the Boundary and Connection families showed that this recovered layer consistently matched the Atlas's own mandatory Places field.

This was not inferred from wording. It was recovered independently, then compared against repository evidence.

## Evidence

Across the audited terms:

- Boundary
- Availability
- Strain
- Bearing
- Resolution
- Connection
- Carry
- Trace
- Carrying
- Recarry
- Coupling

the recovered mechanic and the existing Places field described the same recovered condition. No counterexample was found.

## Recovery

The recovery method therefore does not introduce a new "Mechanic" layer. Instead it adopts the Atlas's existing structural language.

The recovery chain becomes:

```text
Dependency
↓
Contribution
↓
Generation
↓
Places
↓
Reads
↓
Term
```

where:

- Dependency recovers what must remain true.
- Contribution recovers what each dependency contributes.
- Generation recovers the smallest sufficient contribution-set.
- Places states the recovered condition.
- Reads states how that condition becomes recognisable.
- Term names the recovered read.

## Important clarification

This recovery does not change the methodology. It changes the terminology.

The methodology independently recovered a layer already present within the Atlas. The Atlas proved to have stronger language than the temporary working terminology. Therefore Places replaces Mechanic in future methodological work.

## Status

Recovered through reconciliation. No repository change required. No Atlas term changed. Only the working methodology becomes more answerable to the Atlas.

## Recovery Principle

Whenever the recovery method independently rediscovers a structural layer already present in the Atlas, the Atlas terminology takes precedence over the temporary recovery terminology.

The purpose of the methodology is not to create a second language.

Its purpose is to become increasingly answerable to the Atlas itself.
