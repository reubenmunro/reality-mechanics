# P-005 — Weave / Fabric Investigation

**Programme:** Research / architecture  
**Type:** Investigation (no renderer changes, no Atlas/D1 edits, no new public surfaces)  
**Date:** 2026-07-07 (UTC+12)  
**Governing evidence:** P-001, P-002, P-003 (`docs/runtime/INVARIANT_RUNTIME_CONTRACT.md`), P-004 (`docs/reports/P-004-ratio-phase-engine-investigation.md`)  
**Core hypothesis (tested):** Relation does not exist inside a pre-existing medium. Relation, constrained by invariant ratios, weaves the medium. Fabric is the held whole produced by that weaving. Physics may describe fabric behaviour; physics is not the ontology.

**Method:** Atlas frontmatter and `conditions` blocks parsed via repository files (`Reality_Mechanics/**/*.md`, `atlas-core.mjs`, D-008 term register). Runtime cross-check only where Observatory already implements weave-adjacent mechanics. Memory and metaphor are not authority.

## Verdict

| Question | Result |
|---|---|
| Is weave/fabric structurally grounded RM language? | **Yes — partially** |
| Is **Weave** a grounded Atlas term? | **No** — process language only inside Fabric/Web pairing prose |
| Is **Fabric** the higher-order held-continuity term? | **Yes** — `higher.fabric`, faithful D (D-008) |
| Is **Web** the trace-pattern while Fabric is the face? | **Yes** — explicit paired definition in Atlas |
| Should Observatory retire **membrane** as primary medium language? | **Yes** — replace with weave/fabric **hybrid** (Atlas ontology + P-004 field engine) |
| Is commission dependency order exact? | **No** — reorder required (§2) |

**Outcome:** Weave/fabric is **not merely an attractive metaphor**. **Fabric**, **Web**, and **Thread** are published, grounded Atlas terms with explicit dependency edges. **Weave** is the **unregistered verb** those terms already assume. Observatory should adopt **fabric/weave** as medium vocabulary and demote **membrane** to retired physics gloss — without conflating **higher Fabric** with **Atlas Fabric** (practice homonym).

**Next Observatory direction (evidence-backed):**

> The Observatory should let participants see the **weave becoming fabric**, and **fabric expressing behaviour**.

Qualified: “weave” is shown as **thread crossings under declared relation and ratio constraint** (L1–L4), not as a decorative animation. “Fabric” is the **held whole read** (L3–L6), not an elastic sheet.

## 2. Dependency order assessment

### 2.1 Commission-proposed order (tested)

```text
Structure → Relation → Ratio → Thread → Weave → Fabric → Behaviour → Appearance
```

### 2.2 Atlas-backed order (smallest evidence)

```text
L0  Structure invariant (atlasSource, conditions.*)
    ↓
    Relation (primitive — not produced by Ratio)
    ↓
L1  Declared relation facts (holds, traces, carries, pairs, nests)
    ↓
    Connection ← Relation
    Hold ← Resolution
    Carry / Trace ← Hold + Connection
    Read, Term ← Trace/Carry chain
    ↓
    Thread ← Term + Trace + Read          [first-order carrier]
    Ratio ← One + Other + Clearance       [readable form — parallel read arm]
    ↓
    Carrying, Pressure, Coupling, Rhythm  [second-order participation]
    Field of Participation → Organic Field → Field Relationships  [third-order beds]
    Nested Carrying → Invisible/Hidden Bearing
    Structural Gathering → Fabric (higher)  [woven held whole]
    ↓
    Web ← Fabric (trace of crossings)     [carried by Fabric, not before it]
    ↓
L2  Scalar ratio register (runtime mass.carriers, ratioMode)
L3  Parallel structure reads (maturity, order depth, frame membership)
L4  Deterministic behaviour (ratioVisualScale, rhythms, field bend)
    ── cross-cut: frame / visibility
L5  runtimeOverlay
L6  Appearance (Surface / Visible reads — not structure)
```

**Corrections to proposed order:**

| Proposed step | Correction |
|---|---|
| Ratio → Thread | **Wrong direction** — Thread traces to Term/Trace/Read; Ratio is a **parallel read arm** from One/Other/Clearance, not Thread’s parent |
| Thread → Weave | **Weave is not a layer** — implied process between thread accumulation and Fabric |
| Weave → Fabric | **Fabric has six explicit `needs`** — Thread alone is insufficient; requires Field Relationships, Nested Carrying, Invisible Bearing, Surface, Trace |
| Fabric → Web | **Reversed** — Web is **carried by** Fabric (`Fabric.md` `carries: Web`); Web **holds** Fabric as upstream condition — mutual pair, but Fabric places the **face**, Web the **trace** |
| Behaviour / Appearance | Map to P-003 **L4 / L6**, not Atlas order ranks |

### 2.3 Answer — Question 1

**Does Atlas support a dependency order like the commission stack?**

**Partially.** The spine **Structure → Relation → (carriers) → Thread → (higher convergence) → Fabric → Web** is evidenced. **Weave is not a node.** **Ratio** sits in **reading order**, not as the generative step before Thread. **Behaviour** and **Appearance** are **runtime layers** (P-003), not Atlas order ranks.

## 3. Thread / Weave / Fabric / Web distinction

### 3.1 Atlas definitions (authoritative)

| Term | What it is | What it is not |
|---|---|---|
| **Thread** | “Followable continuity through a readable length” — strand before whole | A visual line, stroke, or canvas segment |
| **Weave** (prose only) | Active crossing of threads into continuity — named as “face/trace of the **weave**” | A published term, decoration, or shader effect |
| **Fabric** | “Relation carried as woven continuity… **face of the weave as held whole**” | Membrane, elastic sheet, pre-existing medium, physics body |
| **Web** | “Connected crossing-pattern through which threads remain followable **within** held continuity” | Generic network graph; not the face whole |
| **Field** | “Recurring organisation arising through participation” | Fabric; not a container relation sits inside |

### 3.2 Pairing evidence (Question 3)

From `Web.md` and `Fabric.md` `conditions.pairs`:

> **Fabric** names the face of the weave: woven continuity returned as a held whole.  
> **Web** names the trace of the weave: the connected crossing-pattern through which threads remain followable within that continuity.

**Answer:** **Yes — structurally explicit in Atlas.** Web is path-readable crossing structure; Fabric is whether the woven whole holds.

### 3.3 Thread before Fabric

`Thread.md`:

> [[Fabric]] names many threads held together as woven continuity. Thread is the strand that can be followed **before the fabric has to be read as a whole**.

`Organic Field.md`:

> Within an organic field, a thread may be followed **before the field has been organised as a branch**.

**Answer — Question 2:** **Yes.** Fabric is the higher-order name for held continuity of many threads (plus field crossings, nested carrying, surface, trace).
