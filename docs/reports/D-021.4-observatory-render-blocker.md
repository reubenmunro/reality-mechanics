# D-021.4 — Observatory Render Blocker

**Programme:** Observatory  
**Type:** Render Blocker Fix  
**Date:** 2026-07-06 (UTC+12)  
**Governing evidence:** D-020C (root cause), D-020D (partial bootstrap fix)

---

## Symptom

Live Observatory canvas appeared empty / nearly black on first load. Term selection and side panel still worked — data and hit-testing were alive. Issue was **visual rendering / bootstrap state**, not missing API data.

---

## Root cause (D-020C, confirmed)

Default bootstrap previously selected a high-mass hub (Relation), entered focused condensation with `fieldPressure > 0.38` → `endpointOnly`, suppressing neighbour nodes to near-invisible endpoint cores.

D-020D corrected bootstrap to **neutral whole-field mode** when no hash deep-link:

- `homeMode = true`
- `focusId = null`
- No Relation default
- Focused draw path (where `endpointOnly` applies) skipped

**Remaining gap:** `drawHomeField()` drew only faint relation strokes (alpha ~0.02–0.05) with **no term nodes**. On a dark canvas this still read as empty.

---

## Fix

Complete the D-020D whole-field opening by making the home field **visibly drawable**:

| Change | Purpose |
|---|---|
| `drawHomeNode()` | Render visible term cores at deterministic home positions |
| `neutralWholeFieldOpen()` | Single predicate: `homeMode && !focusId` |
| Node budget (`homeNodes: 140`) | Scale term visibility with adaptive render quality |
| Whole-field line boost (×2.6) | Brighter home relation strokes on neutral open only |
| `holds` in `buildHomeConnections()` | Richer whole-field relation graph |
| Bootstrap unchanged in intent | Still no Relation default; hash URLs preserved |

**Not changed:** D-020A compressed panel, D-020B mechanics amplification coefficients, Atlas source, focused condensation after explicit term selection.

---

## Behaviour after fix

| Entry | Canvas | Panel |
|---|---|---|
| `/field` (no hash) | Whole field: visible nodes + relations | Neutral Observatory panel |
| `/field#ground.invariant` | Focused condensation on Invariant | D-020A term panel |

| Continue where I left off | Restores last term focus | D-020A term panel |

`endpointOnly` remains inactive on neutral open because the focused draw branch (`if (!homeMode)`) does not run.
