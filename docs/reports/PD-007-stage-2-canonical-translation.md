- Atlas Markdown structural parsing;
- order rankings and order/register declarations;
- relation vocabulary;
- protocol declarations;
- surface-role declarations;
- canonical Theory prose;
- structural source-format documentation.

Bounded product selections remain maintained only where they control rendering or evidence. They are labelled non-canonical and validate every selected canonical ID or relation key against generated data.

## 3. MCP migration

MCP now imports only the generated participation module plus its generated D1 read model.

Removed:

- maintained protocol IDs;
- `startingIds`;
- the legacy four-entry protocol;
- maintained layer/operator/output contracts;
- maintained surface-role tools;
- per-entry source-hash claims.

The MCP protocol is the generated ordered `ai-entry` protocol. Session start fails if a member is missing or D1 source identity differs from the generated source identity.

MCP reads all seven generated relation types and returns generated register, order, determination, conditions, and protocol data.
