# Calibration

Calibration is a public reasoning instrument for Reality Mechanics.

It is not AI. It does not answer for the participant, and it does not need the participant's input. It shows that reasoning has mechanics by running one, live, in front of them.

```text
Target drifts -> Strain builds -> Threshold crosses -> Pulse fires -> Approximation corrects -> Carried Strain remains
```

## What It Does

- Runs a target that drifts on its own — nothing holds it still.
- Tracks Open Strain, the live gap between the target and the current approximation, as a scrolling cardiogram trace read against the mechanism's own local average.
- Fires a Pulse — a discrete correction, not a continuous chase — once strain crosses a threshold.
- Corrects the approximation by an imperfect fraction of the gap, sometimes overshooting, sometimes undershooting.
- Surfaces Carried Strain: the leftover after a pulse, shown explicitly rather than left implicit, because it measurably (if weakly) shapes how soon the next pulse fires.
- Never lets Open Strain settle at zero. The target keeps moving; no pulse closes the whole gap.

## Reading Against Reality Mechanics

Checked directly against the Atlas rather than assumed. The canonical first-order spine is Strain → Bearing → Resolution. Open Strain already matches Strain. The held/rising phase before a pulse is Bearing — strain sustained, not yet collapsed or resolved. Pulse is the *mechanism*; the structural role it performs is Resolution — a discrete event, not continuous control, that settles held strain into a readable outcome without ending the relation.

Resolution is marked order-terminal for First Order in the Atlas: once it fires, "what continues must be carried forward, re-entered, or read under another order." That isn't a design choice Calibration made up — it's what Resolution requires structurally. Carried Strain exists because Resolution's own definition demands something survive it.

## What It Does Not Do

- No text input, no form, no submission.
- No OpenAI or model call.
- No MCP call.
- No Atlas mutation.
- No account, password, passphrase, or hidden admin action.
