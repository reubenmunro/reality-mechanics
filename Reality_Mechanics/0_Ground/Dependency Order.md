# Dependency Order

Structural precedence within Root Order — what must already hold before another term can become available.

## Places

Dependency Order places structural precedence within Root Order. It names what the governing arrangement reveals: one term must already hold before another can become available.

## Holds

Dependency Order is held by [[Root Order]].

## Pairs

Dependency Order pairs with [[Natural Order]]. Dependency Order names structural precedence — what must already hold within the arrangement; Natural Order names the order available within the arrangement without imposition or enforcement. Each requires the other to be locatable: structural precedence is only readable against order that holds naturally; available order is only readable against what must structurally precede it.

## Traces

- [[Root Order]]

## Nests

Dependency Order nests inside [[Root Order]] and opens later dependency reads without becoming chronology, command, or preference.

## Reads

Dependency Order becomes recognisable where a term can be traced through its prior conditions without confusing structural precedence with temporal sequence. It distinguishes what must hold first from what merely appears first.
