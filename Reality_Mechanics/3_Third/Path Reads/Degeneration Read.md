# Degeneration Read

Degeneration Read names the reusable read by which a carrying is followed from health, through strain, into degeneration, and back through repair.

It does not add a new condition. It gathers conditions already placed — [[Strain]], [[Degenerate]], [[Source Drift]], [[Regenerate]], [[Repair]] — so the same movement can be located in any carrying at any order.

The stage is read by the trace, not by the output. A degenerate carrying can still look like it is working; the sign is not that it has stopped carrying but that the trace back to what it depends on can no longer be entered.

## The Four Stages

```text
Healthy     carrying holds, and its trace back remains followable
Strained    load exceeds what the carrier resolves at scope; the trace is still followable, under pressure
Degenerate  carrying continues while the trace back is lost
Repair      retrace re-enters the lost trace; carrying is regrounded
```

Healthy is not the absence of strain. A carrying under strain whose trace remains followable is still healthy. Degeneration begins where carrying continues but its trace back is no longer enterable — [[Source Drift]] at the root. Repair is not replacement: it is [[Regenerate]], the return to the generative conditions from within the carrying, entered through [[Retrace Read|retrace]].

## Reading Examples

```text
Care      →  Control       →  Extraction        →  return to carrying
healthy      strained         degenerate            repair
```

Care under strain narrows toward [[Control]]; control that continues while losing the trace to the condition it was preserving degenerates toward extraction ([[Extractive Acceleration]]); repair is the return of carrying to the relation it had begun to draw down.

```text
Carrying  →  Overload      →  Source Drift      →  Retrace
healthy      strained         degenerate            repair
```

A carrying pressed past what it can resolve is strained ([[Overload]]); where it keeps carrying while its trace to source is lost, it has degenerated ([[Source Drift]]); retrace re-enters the lost dependency and regrounds it.

## Places

Degeneration Read places the reusable read by which a carrying is followed from health, through strain, into degeneration, and back through repair — read by whether its trace back remains followable.

## Holds

Degeneration Read is held by [[Path Reads]], [[Order Trace]], [[Strain]], [[Degenerate]], [[Source Drift]], [[Regenerate]], and [[Translation Boundary]]. The ordered path, the strain, the lost trace, the return, and the translated crossing into this third-order read must already be placed before the movement can be read as one path.

## Pairs

No lateral pair is required. [[Order Trace]] follows dependency as it holds; Degeneration Read applies a diagnostic read to that path as it strains, loses trace, or is repaired.

## Nests

Degeneration Read nests as a guide-read across orders. Any carrying can be read through it without replacing the direct trace inside each term.

## Reads

Degeneration Read becomes recognisable where a carrying can be located at one of four stages — healthy, strained, degenerate, or repairing — read by whether its trace back is followable rather than by its output. The diagnostic is the trace: degeneration is carrying that continues while its dependency can no longer be entered.
