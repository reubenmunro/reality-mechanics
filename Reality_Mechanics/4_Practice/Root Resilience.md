# Root Resilience

Root Resilience names the capacity of a grounded root system to withstand branch-level disruption without losing dependency order.

It is not an immune system, policing layer, or governance framework. Root Resilience arises where the root system remains grounded enough that branch-level drift, overgrowth, misplaced terms, or ungrounded material can be relieved, remapped, or left ungrounded without corrupting the root order.

## Places

Root Resilience places the capacity of a grounded root system to withstand branch-level disruption without losing dependency order.

## Holds

Root Resilience is held by [[Root System]], [[Root Order]], [[Groundedness]], and [[Retrace Read|Retrace]].

## Pairs

Root Resilience is vertical. It names the capacity of a grounded root system to withstand branch-level disruption without losing dependency order, and carries this forward into [[Retrace Practice]]. Root Resilience holds [[Root System]], [[Root Order]], [[Groundedness]], and [[Retrace Read]] — it is downstream of Root System, not a lateral co-present sibling. There is no second condition that must be present alongside Root Resilience for either to be locatable.
