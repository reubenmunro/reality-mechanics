# Shade Clearing

Shade Clearing names the garden practice of restoring readability where order is present but shaded by crowding, residue, missing trace, misplaced support, or overgrowth.

Shade is not badness and not a rival substance. Shade is order under occlusion. Clearing shade does not attack the growth; it restores the conditions by which the growth, its support, and its effects can be read.

## Places

Shade Clearing places the garden practice of restoring readability where order is present but shaded by crowding, residue, missing trace, misplaced support, or overgrowth.

## Holds

Shade Clearing is held by [[Garden Pass]], [[Structural Disorder]], [[Clear]], and [[Clean]]. A tending pass must locate the shaded growth, structural disorder must name the loss of dependency readability, clear must make the path readable, and clean must keep the correction from carrying residue.

## Pairs

No lateral pair is required. Shade Clearing is a care action inside Garden Pass.

## Nests

Shade Clearing nests inside Atlas Garden wherever order is present but cannot be read cleanly enough for re-entry.

## Reads

Shade Clearing becomes recognisable where correction removes obscuring growth, residue, missing trace, or misplaced support so an order can be re-entered without treating shade as a separate thing.

The clearing question is:

```text
What order is present here?
What prevents it from being read?
Is the shade caused by missing trace, crowding, residue, misplaced support, or overgrowth?
What can be pruned, grafted, spaced, composted, or left working so re-entry returns?
```

## Carries

No downstream carries are claimed yet. Shade Clearing is a care action; later carries should be added only where a repeated clearing practice becomes a distinct retraceable continuation.
