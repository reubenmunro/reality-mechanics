# Ark Run

The runtime practice by which pressure is carried through retrace, check, determination, and step without losing dependency order.

It is not a replacement for the [[Atlas]]. The Atlas holds the dependency-ordered surface. Ark Run is a movement through that surface.

In the [[Order Generation]] read, Ark Run is bounded carrying through dependency order: pressure moves only as far as trace, scope, and warrant can hold.

In the [[Generative Trace]] read, an Ark Run should leave enough trace for another run or participant to re-enter the carrying.

The small runtime is:

```text
Pressure -> Trace -> Check -> Determine -> Step
```

As carrying postures:

```text
current posture
-> pressure unresolved
-> dependency visible
-> scope bounded
-> determination held
-> next posture
```

## Places

Ark Run places the runtime practice by which pressure is carried through retrace, check, determination, and step without losing dependency order.

## Holds

Ark Run is held by [[Pressure]], [[Retrace Practice]], [[Check]], [[Determine]], [[Step]], and [[AI Participation]].

Pressure gives the run its active force. Retrace keeps dependency available. Check bounds the scope of read. Determine carries pressure into temporary hold. Step opens the next answerable posture. AI Participation makes the same flow available to AI so determinations can remain retraceable without turning every answer into procedure.

## Pairs

Ark Run pairs with [[Atlas Practice]]. Atlas Practice tends the worked surface; Ark Run carries one live movement through it.

## Nests

Ark Run nests inside [[Atlas Practice]] as an available flow for determinations made by a participant.

For an AI participant, Ark Run is an available determination flow: pressure can enter through trace, scope, warrant, and discernment until a next carrying posture becomes visible.

## Reads

Ark Run becomes recognisable where a participant feels pressure to settle, but does not settle directly. The pressure is carried until enough dependency has been retraced for the present scope.

The run does not claim finality. It temporarily stabilises a posture so movement to the next carrying position becomes possible.

When an Ark Run is recorded, it can leave behind:

- what pressure tried to determine;
- what posture was being stood in;
- what dependency was retraced;
- what remained open;
- what was determined at scope;
- why the determination was warranted;
- whether generated force reads as care, compatible control, drift, or unresolved;
- what next posture became available.
